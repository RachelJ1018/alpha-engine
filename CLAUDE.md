# Alpha Engine ‚Äî CLAUDE.md

Personal AI-powered equity research tool. Runs daily, scores 31 watchlist symbols,
generates trade theses, tracks paper trade outcomes, and sends notifications.

## Stack
- Python 3.9 (macOS local) / Python 3.12 (GitHub Actions)
- SQLite (`data/alpha.db`) ‚Äî single persistent store
- yfinance ‚Äî price data + options chain
- feedparser + requests ‚Äî news (RSS + Yahoo Finance)
- Streamlit ‚Äî local dashboard (`app.py`)
- GitHub Actions ‚Äî daily cron (Mon‚ÄìFri 7:30am PT)

## Daily Pipeline (run_daily.py)
1. `init_db()` / `migrate_db()` ‚Äî create/migrate schema
2. `collect_news()` ‚Äî fetch RSS articles, deduplicate
3. `fetch_prices()` ‚Äî OHLCV + RSI/MA/ATR + **day_high/day_low** for all symbols
4. `get_market_regime()` ‚Äî SPY change ‚Üí bull/bear/neutral/choppy
5. `update_outcomes()` ‚Äî resolve PENDING signals using fresh prices (intraday high/low)
6. `run_analysis()` ‚Äî score all symbols, generate theses
7. `record_signals()` ‚Äî save ACTIONABLE/WATCHLIST/MONITOR to signal_outcomes
8. `generate_report()` ‚Äî write MD + HTML to reports/
9. `send_daily_notification()` ‚Äî email + WeChat (skipped if no env vars)

## 4-Layer Scoring Architecture (analyzer.py)

### Score formula (max raw = 85, not 100)
```
final_score = EventEdge(0-25) + MarketConf(0-20) + RegimeFit(0-15)
            + RelOpp(0-15) + Freshness(0-10) - RiskPenalty(0-15)
```

### Action labels ‚Äî empirical multi-factor tiers
Thresholds are anchored to actual historical outcomes (score range 44‚Äì66),
not to the theoretical max of 85. Regime influence lives in the RegimeFit
scoring layer, not in threshold shifts.

| Tier | Condition |
|------|-----------|
| **IGNORE** | score < 50, **or** bucket in {macro_watch, sympathy_play, opinion_watch} |
| **MONITOR** | round(score,1) 50‚Äì51, or pos_mult < 0.55 |
| **WATCHLIST** | round(score,1) ‚â• 52 + quality gates pass |
| **ACTIONABLE** | WATCHLIST gates + EventEdge ‚â• 12 + **one of**: |
| | a. post_earnings_drift + vr 1.0‚Äì1.5 + MC 10‚Äì15 |
| | b. earn_strength ‚â• 3 (strongly confirmed earnings gap) |
| | c. score ‚â• 58 + alpha-positive bucket + MC ‚â• 12 |

Quality gates (required for WATCHLIST+): bucket not excluded, pos_mult ‚â• 0.55.
Alpha-positive buckets: `post_earnings_drift`, `event_long`, `event_short`.
Score is rounded to 1 dp before comparison so displayed value matches tier.

Empirical validation (124 resolved signals, Mar‚ÄìMay 2026):
- ACTIONABLE n=7 ‚Üí win 71.4%, avg t+5 +2.56%
- WATCHLIST  n=48 ‚Üí win 60.4%, avg t+5 +1.36%
- IGNORE     n=59 ‚Üí win 62.7%, avg t+5 +1.04%

### Key scoring details
- **EventEdge**: source quality + event strength + novelty + options flow boost (‚àí2 to +3)
  - Earnings signals: `score_post_earnings_risk()` adds `earn_strength` bonus (0‚Äì4.5) for large
    confirmed gaps (direction-aligned, vr‚â•1.0, gap held intraday, regime not counter)
- **MarketConf**: price/MA/volume alignment with thesis direction
- **RegimeFit**: bull‚ÜíLONG gets 11, bear‚ÜíSHORT gets 10.5; never hard-blocks
- **RelOpp**: strategy-aware 52-week distance scoring; macro_watch returns flat 5.0
- **RiskPenalty**: RSI event-type aware (no penalty for SHORT+RSI>70 or event-driven LONG+RSI>75);
  earnings confirmation gate floors penalty at 3.5/2.0/1.0 based on n_confirmed (1‚Äì3 of 4 criteria)
- **Direction**: news sentiment ‚Üí price trend fallback
- **earn_strength**: stored in `trade_candidates`; used as ACTIONABLE edge indicator (condition b)
- **position_size_mult**: stored in `trade_candidates`; choppy‚Üí0.65, bear+LONG‚Üí0.85, high-ATR‚Üí√ó0.85

### Options Flow Boost (options_flow.py)
Applied to EventEdge after news scoring, before no-news cap. Range: ‚àí2.0 to +3.0.
- Call/Put volume ratio from nearest 7‚Äì45 day expiry (yfinance options chain)
- Vol/OI ratio flags unusual institutional activity (+0.5 if vol > 50% of OI)
- ETFs excluded (SPY/QQQ/IWM/SMH/GLD/TLT/XLF) ‚Äî their CP ratio reflects hedging
- Cached per symbol per day (fetched once, direction boost computed fresh per call)

## Multi-Agent Thesis (multi_agent_thesis.py)
4-agent pipeline: TechnicalAgent ‚Üí NewsAgent ‚Üí RiskAgent ‚Üí SynthesisAgent.

Activated by `THESIS_PROVIDER=multi_agent`. Stores per-run in `trade_candidates`:
- `thesis_conviction` ‚Äî HIGH / MEDIUM / LOW
- `thesis_technical`, `thesis_news`, `thesis_risk` ‚Äî individual agent reports (‚â§1000 chars)

Gemini uses `streamGenerateContent` (SSE streaming) to prevent mid-JSON truncation.
Agent max_tokens: 800 per agent, 1500 for synthesis. Agent reports trimmed to 500 chars before synthesis to keep prompt focused.

### Multi-Provider Thesis (THESIS_PROVIDER env var)
- `auto` ‚Äî Anthropic Haiku (WATCHLIST/MONITOR), Sonnet (ACTIONABLE); falls back Groq ‚Üí Gemini ‚Üí rule-based
- `google` ‚Äî Gemini Flash always (free, 1500 req/day)
- `groq` ‚Äî Llama 3.3 70B (free, 14400 req/day)
- `anthropic` ‚Äî Anthropic only
- `multi_agent` ‚Äî 4-agent pipeline (uses provider internally, default auto)
- `none` ‚Äî rule-based fallback only

## Signal Tracking & Paper Trades (signal_tracker.py)
- `record_signals()` ‚Äî INSERT OR IGNORE (safe for re-runs same day)
- `update_outcomes()` ‚Äî fills t+1/t+3/t+5 P&L from accumulated price_snapshots
- Paper trade model: stop = entry ¬± 1.2√óATR, target = entry ¬± 2√óstop (2R)
- **Stop/target detection uses intraday day_high/day_low** (more accurate than EOD close)
  - Falls back to close_price for older rows where day_high/day_low is NULL
- Exit: HIT_STOP / HIT_TARGET / T5_EXIT (whichever comes first within 5 days)

## DB Schema (key tables)
- `watched_symbols` ‚Äî 31 symbols with sector + priority flags
- `news_articles` ‚Äî deduplicated by title hash; stores event_type, sentiment, novelty
- `price_snapshots` ‚Äî one row per symbol per day; includes atr_14, week_high_52, **day_high, day_low**
- `trade_candidates` ‚Äî scored output per run; includes 5-component scores + **thesis_conviction/technical/news/risk** + **earn_strength, position_size_mult**
- `signal_outcomes` ‚Äî forward P&L tracking; t1/t3/t5 + paper_exit + paper_pnl
- `daily_runs` ‚Äî run log

## Weight Optimizer (weight_optimizer.py)
Computes Pearson correlation between each layer score and t+5 P&L.
```bash
python -m modules.weight_optimizer          # preview
python -m modules.weight_optimizer --apply  # write data/weights.json
```
Requires 3+ months of resolved outcomes to be meaningful.
`analyzer.py` loads `data/weights.json` on startup if it exists.

## Backtest (backtest_runner.py)
Price-only (no historical news). Downloads 90 days via yfinance, reconstructs
daily price rows, uses fixed EventEdge=15 proxy.
```bash
python3 backtest_runner.py
python3 backtest_runner.py --csv    # saves data/backtest.csv
```
Key finding: price layers alone ‚Üí ~49% win rate (expected ‚Äî EventEdge is the real edge).
