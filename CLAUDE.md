# Alpha Engine — CLAUDE.md

Personal AI-powered equity research tool. Runs daily, scores 31 watchlist symbols,
generates trade theses, tracks paper trade outcomes, and sends notifications.

## Stack
- Python 3.9 (macOS local) / Python 3.12 (GitHub Actions)
- SQLite (`data/alpha.db`) — single persistent store
- yfinance — price data + options chain
- feedparser + requests — news (RSS + Yahoo Finance)
- Streamlit — local dashboard (`app.py`)
- GitHub Actions — daily cron (Mon–Fri 7:30am PT)

## Daily Pipeline (run_daily.py)
1. `init_db()` / `migrate_db()` — create/migrate schema
2. `collect_news()` — fetch RSS articles, deduplicate
3. `fetch_prices()` — OHLCV + RSI/MA/ATR + **day_high/day_low** for all symbols
4. `get_market_regime()` — SPY change → bull/bear/neutral/choppy
5. `update_outcomes()` — resolve PENDING signals using fresh prices (intraday high/low)
6. `run_analysis()` — score all symbols, generate theses
7. `record_signals()` — save ACTIONABLE/WATCHLIST/MONITOR to signal_outcomes
8. `generate_report()` — write MD + HTML to reports/
9. `send_daily_notification()` — email + WeChat (skipped if no env vars)

## 4-Layer Scoring Architecture (analyzer.py)

### Score formula (max raw = 85, not 100)
```
final_score = EventEdge(0-25) + MarketConf(0-20) + RegimeFit(0-15)
            + RelOpp(0-15) + Freshness(0-10) - RiskPenalty(0-15)
```

### Action labels (regime-aware thresholds)
| Score      | LONG (neutral) | SHORT (neutral) |
|------------|---------------|-----------------|
| ACTIONABLE | ≥ 77          | ≥ 74            |
| WATCHLIST  | ≥ 62          | ≥ 60            |
| MONITOR    | ≥ 48          | ≥ 48            |
| IGNORE     | < 48          | < 48            |

Thresholds shift by regime: bear raises LONG threshold, lowers SHORT threshold.

### Key scoring details
- **EventEdge**: source quality + event strength + novelty + options flow boost (−2 to +3)
- **MarketConf**: price/MA/volume alignment with thesis direction
- **RegimeFit**: bull→LONG gets 11, bear→SHORT gets 10.5; never hard-blocks
- **RelOpp**: 52-week high distance (+4 if >30% off high, -2 if <3% off high)
- **RiskPenalty**: RSI counter-trend (SHORT+RSI>70 → +2.5), ATR%, gap risk
- **Direction**: news sentiment → price trend fallback

### Options Flow Boost (options_flow.py)
Applied to EventEdge after news scoring, before no-news cap. Range: −2.0 to +3.0.
- Call/Put volume ratio from nearest 7–45 day expiry (yfinance options chain)
- Vol/OI ratio flags unusual institutional activity (+0.5 if vol > 50% of OI)
- ETFs excluded (SPY/QQQ/IWM/SMH/GLD/TLT/XLF) — their CP ratio reflects hedging
- Cached per symbol per day (fetched once, direction boost computed fresh per call)

### Strategy buckets
`event_short`, `event_long`, `sympathy_play`, `post_earnings_drift`, `macro_watch`

## Multi-Agent Thesis (multi_agent_thesis.py)
4-agent pipeline: TechnicalAgent → NewsAgent → RiskAgent → SynthesisAgent.

Activated by `THESIS_PROVIDER=multi_agent`. Stores per-run in `trade_candidates`:
- `thesis_conviction` — HIGH / MEDIUM / LOW
- `thesis_technical`, `thesis_news`, `thesis_risk` — individual agent reports (≤1000 chars)

Gemini uses `streamGenerateContent` (SSE streaming) to prevent mid-JSON truncation.
Agent max_tokens: 800 per agent, 1500 for synthesis. Agent reports trimmed to 500 chars before synthesis to keep prompt focused.

### Multi-Provider Thesis (THESIS_PROVIDER env var)
- `auto` — Anthropic Haiku (WATCHLIST/MONITOR), Sonnet (ACTIONABLE); falls back Groq → Gemini → rule-based
- `google` — Gemini Flash always (free, 1500 req/day)
- `groq` — Llama 3.3 70B (free, 14400 req/day)
- `anthropic` — Anthropic only
- `multi_agent` — 4-agent pipeline (uses provider internally, default auto)
- `none` — rule-based fallback only

### Env vars
```bash
GOOGLE_API_KEY       # Gemini (gemini-3-flash-preview)
GROQ_API_KEY         # Groq
ANTHROPIC_API_KEY    # Anthropic (needs billing credits)
THESIS_PROVIDER      # default: "auto"
```

## Signal Tracking & Paper Trades (signal_tracker.py)
- `record_signals()` — INSERT OR IGNORE (safe for re-runs same day)
- `update_outcomes()` — fills t+1/t+3/t+5 P&L from accumulated price_snapshots
- Paper trade model: stop = entry ± 1.2×ATR, target = entry ± 2×stop (2R)
- **Stop/target detection uses intraday day_high/day_low** (more accurate than EOD close)
  - Falls back to close_price for older rows where day_high/day_low is NULL
- Exit: HIT_STOP / HIT_TARGET / T5_EXIT (whichever comes first within 5 days)

## DB Schema (key tables)
- `watched_symbols` — 31 symbols with sector + priority flags
- `news_articles` — deduplicated by title hash; stores event_type, sentiment, novelty
- `price_snapshots` — one row per symbol per day; includes atr_14, week_high_52, **day_high, day_low**
- `trade_candidates` — scored output per run; includes 5-component scores + **thesis_conviction/technical/news/risk**
- `signal_outcomes` — forward P&L tracking; t1/t3/t5 + paper_exit + paper_pnl
- `daily_runs` — run log

## Weight Optimizer (weight_optimizer.py)
Computes Pearson correlation between each layer score and t+5 P&L.
```bash
python -m modules.weight_optimizer          # preview
python -m modules.weight_optimizer --apply  # write data/weights.json
```
Requires 3+ months of resolved outcomes to be meaningful.
`analyzer.py` loads `data/weights.json` on startup if it exists.

## Backtest (backtest_runner.py)
Price-only (no historical news). Downloads 90 days via yfinance, reconstructs
daily price rows, uses fixed EventEdge=15 proxy.
```bash
python3 backtest_runner.py
python3 backtest_runner.py --csv    # saves data/backtest.csv
```
Key finding: price layers alone → ~49% win rate (expected — EventEdge is the real edge).
